# Para League Newsroom Product Model

The app is a generation-first newsroom shell.

Core loop:

structured league data -> context packet/story plan -> Gemini generation -> editable admin draft -> approved/published public text

Public pages do not generate copy. Public pages show approved/published text or a polished placeholder.

Admin pages generate, inspect context, inspect included docs, edit, save, publish, and unpublish drafts.

The product should feel like a small competitive league with an editorial desk, not a dashboard, debug archive, or raw stat export.
